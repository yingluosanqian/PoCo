# Need

## 背景

在 card-first 平台无关骨架已经落地后，需要把它推进到一条最小可跑的 DM 卡片链，否则仍然无法验证 project 管理侧的真实交互。

## 需求信号

- 需要从纯协议对象进入最小业务链路
- 需要至少有一条 DM project 管理卡片能跑通

## 来源

- 用户要求继续推进实现

## 场景

- 在 DM 中查看 project 列表
- 在 DM 中通过 card action 创建 project
- 在 DM 中打开 project 详情

## 频率/影响

这是 card-first 从抽象进入真实交互的第一条链路。

## 备注

当前目标仍是最小链路，不是完整 card 体系。

# Need

## 背景

PoCo 希望服务移动场景下的开发工作。发起人明确提出，希望通过手机去控制运行在服务器上的 AI agent，而不是被桌面环境绑定。

## 需求信号

- 在离开电脑时，仍希望能继续推动真实开发任务
- 希望通过熟悉的消息入口与远端 agent 交互
- 希望 agent 在服务器侧持续运行，而不是依赖手机本地能力

## 来源

- 项目发起人直接提出的产品方向

## 场景

- 外出、通勤、临时离开工位时，通过手机发起任务
- 在移动端查看 agent 当前进展、阻塞和结果
- 当任务需要人工确认时，直接在消息入口中处理

## 频率/影响

该需求不是一次性偏好，而是项目存在的核心前提。如果不能在移动场景下完成对远端 agent 的有效控制，PoCo 的核心价值就不成立。

## 备注

飞书 bot 和 Slack bot 都被提出过，但它们首先是候选接入方式，不自动等于项目问题定义。

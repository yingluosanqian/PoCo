# Problem

## 背景

PoCo 的目标是让开发者在移动场景下，也能控制运行在服务器上的 AI agent 持续完成开发任务。

## 相关需求

- 通过手机发起远端 agent 任务
- 通过消息入口查看进展和结果
- 在需要时完成人工确认

## 当前状态

- 当前仓库只有开发治理结构，没有任何产品实现
- 当前没有可用的消息入口接入
- 当前没有服务器侧 agent 任务管理、状态回传或确认机制
- 当前用户无法在移动端完成一次真实的远端 agent 工作流

## 问题定义

PoCo 当前最值得优先解决的问题是：

开发者在移动场景下，缺少一个低摩擦、可控、可持续的方式，去发起、跟进并必要时确认服务器侧 AI agent 的真实工作流。

## 为什么这是个真实问题

- 这直接阻碍了项目目的成立；如果用户必须回到桌面端才能有效控制 agent，PoCo 的核心价值就失效
- 当前缺的不是某个局部 UI，而是完整主链路不存在
- 这是后续平台接入、任务模型、确认机制等决策的共同上游问题

## 不是什么问题

- 不是“先支持多少个平台”的问题
- 不是“先做飞书还是 Slack”本身的问题
- 不是“在手机上写完整代码”的问题
- 不是“做一个通用聊天机器人平台”的问题

## 证据

- 项目目的已明确要求服务移动场景下的远端 agent 控制
- 当前状态已明确显示，现有仓库不具备任何可运行工作流
- 发起人已明确提出手机控制服务器侧 agent 的核心诉求

# Plan

## 目标

定义卡片后端交互的最小协议，使 PoCo 的卡片实现具备统一动作入口和可预测刷新行为。

## 范围

- 定义 action 意图协议
- 定义最小刷新模式
- 定义哪些卡片和动作使用哪些刷新模式

## 不在范围内的内容

- 飞书 schema 级字段定义
- 卡片组件细节实现
- 完整错误处理策略

## 风险点

- 容易把协议设计过复杂
- 容易让刷新模式和业务状态强耦合
- 容易遗漏 DM 和群之间的差异

## 验收标准

- 已明确 action id 不等于按钮文案
- 已明确固定刷新模式集合
- 已明确主要卡片动作对应的刷新策略
- 可支撑下一轮实现设计

## 实施顺序

1. 固化 need/problem/decision
2. 形成协议与刷新设计
3. 回写稳定层设计摘要
4. 下一轮再进入 callback handler 与 view model 的实现设计

# Problem

## 当前状态

- `agent = project identity`
- `working dir = session stance`

这两个原则已经成立，但卡片层还没有回答：

- 用户在 DM 里应该先看到什么
- 用户在群里应该先看到什么
- 哪些动作应该一级可见，哪些动作应该收进二级面板

## 问题定义

PoCo 当前缺少一套与上下文 ownership 一致的卡片信息架构，导致 `agent` 和 `working dir` 虽然在概念上已被区分，但在实际交互中仍可能重新混淆。

## 不是的问题

- 不是现在就实现所有配置持久化的问题
- 不是现在就设计完整 repo 浏览器的问题
- 不是所有高级动作都要进入首屏卡片的问题

# Decisions Layer

## 本层定义

本层负责在多个问题或多个方向之间做取舍。

它定义这一轮优先做什么，不做什么，以及为什么这样选。

## 本层关注的问题

- 当前轮最值得投入的方向是什么
- 哪些方案被放弃
- 为什么做这个，而不是别的
- 本轮明确不做什么

## 输入

- 已成立的问题
- 约束条件
- 当前资源与风险预算
- 候选方案比较

## 输出

- 决策记录
- 本轮优先项
- 本轮非优先项
- 取舍理由

## 禁止事项

- 不要把“什么都做”写成决策
- 不要只记录结论，不记录放弃理由
- 不要绕开约束做“战略性例外”
- 不要在实现阶段补做未被决策批准的内容

## 进入下一层的条件

满足以下条件后，才能进入计划层：

- 已明确本轮做什么
- 已明确本轮不做什么
- 取舍理由可被复述
- 风险和代价至少有最小说明

## 决策记录最少应包含

- 待选问题或方案
- 当前决策
- 为什么这样选
- 为什么不选其他方案
- 风险
- 后续影响

## PoCo 当前决策

### 第一阶段入口决策

第一阶段优先接飞书 bot，作为 PoCo 第一条可运行主链路的消息入口。

### 为什么这样选

- 发起人当前已明确要求优先接飞书
- 单平台优先有利于压缩范围和验证主链路
- 这符合当前约束：先跑通、再扩展；先真实可用、再泛化

### 当前明确不做

- 不同时支持飞书和 Slack
- 不先做通用平台抽象层
- 不把消息平台数量当成当前阶段成功标准

### 当前实现栈取向

当前实现层优先使用 Python 搭建服务端骨架。

### 当前 agent 优先级

当前执行器方向计划包含 Codex、Claude Code、Cursor Agent。

当前优先实现 Codex。

### 当前上下文责任边界决策

产品级连续性交接上下文由 PoCo 持有。

执行期上下文由 Codex、Claude Code、Cursor Agent 等 backend 各自持有。

当前不把“完整上下文”全部外包给单一 backend，也不把 PoCo 扩写成重型 memory 系统。

### 当前正式交互模型决策

PoCo 的正式交互模型调整为：

- 单聊负责 project 管理与建群绑定
- 群负责 project 的正式执行与协作

也就是：

- `DM -> control plane`
- `Group -> project workspace`

project 之下继续承载 session 和 task。

### 当前交互协议决策

PoCo 的正式交互主路径转向飞书卡片 2.0。

文本命令不再作为正式用户协议，只在过渡期保留为开发、联调或故障回退路径。
